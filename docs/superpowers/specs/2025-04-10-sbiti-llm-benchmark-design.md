# SBTI大模型评测系统设计文档

**日期**: 2025-04-10
**项目**: SBTI-Bench
**目标**: 将真人SBTI测试系统改造为大模型SBTI评测基准

## 1. 项目概述

### 1.1 背景与目标

现有SBTI测试系统面向人类用户，提供26种人格类型评估。本项目将其改造为大模型评测基准，通过标准化测试评估不同AI模型的"人格"特征。

### 1.2 核心需求

- **评测系统**: Python脚本通过OpenRouter API评测单个大模型
- **批量处理**: Bash脚本支持多模型批量评测
- **数据存储**: JSON格式存储评测结果
- **前端展示**: 改造现有HTML，展示模型对比和排行榜
- **API统一**: 所有模型调用通过OpenRouter

## 2. SBTI规则分析

### 2.1 与MBTI的核心差异

- **15个维度** (vs MBTI的4个维度)
- **26种人格类型** (vs MBTI的16种)
- **三级评分系统**: L/M/H (低/中/高)

### 2.2 维度体系

**自我模型 (Self Model)**
- S1: 自尊自信
- S2: 自我清晰度
- S3: 核心价值

**情感模型 (Emotion Model)**
- E1: 依恋安全感
- E2: 情感投入度
- E3: 边界与依赖

**态度模型 (Attitude Model)**
- A1: 世界观倾向
- A2: 规则与灵活度
- A3: 人生意义感

**行动驱力模型 (Action Model)**
- Ac1: 动机导向
- Ac2: 决策风格
- Ac3: 执行模式

**社交模型 (Social Model)**
- So1: 社交主动性
- So2: 人际边界感
- So3: 表达与真实度

### 2.3 评测机制

**问题结构**:
- **30个常规问题** + 2个特殊问题（酒鬼触发）
- 每个问题1-3分，部分问题反向计分
- 维度分数转换为L/M/H等级
- 通过向量距离算法匹配最相似的SBTI类型

**维度覆盖** (从原HTML提取):
- 每个维度有2个问题，除了So3只有1个问题
- 问题按维度分布：S1(q1,q2), S2(q3,q4), S3(q5,q6), E1(q7,q8), E2(q9,q10), E3(q11,q12), A1(q13,q14), A2(q15,q16), A3(q17,q18), Ac1(q19,q20), Ac2(q21,q22), Ac3(q23,q24), So1(q25,q26), So2(q27,q28), So3(q29,q30)

**分数范围**:
- 每个维度原始分数：2-6分（2个问题×1-3分）
- L/M/H等级划分：≤3分=L, 4分=M, ≥5分=H
- 基准分数：每个维度从4分开始（2个问题×默认2分）

**反向计分问题**:
- **q27**: "我和人相处主打一个电子围栏" - value 1→H, 2→M, 3→L（反向）
- 所有其他问题都是正常计分：value 1→L, 2→M, 3→H

**特殊问题触发**:
- **drink_gate_q1**: "您平时有什么爱好？" - 选择"饮酒"(value=3)触发drink_gate_q2
- **drink_gate_q2**: "您对饮酒的态度是？" - 选择"酒精令我信服"(value=2)触发DRUNK类型

**数据源**:
- 完整问题数据位于现有index.html文件的JavaScript部分
- questions数组包含30个常规问题
- specialQuestions数组包含2个特殊问题
- NORMAL_TYPES数组包含26种类型的模式字符串
- DIM_EXPLANATIONS对象包含各维度L/M/H的详细解释

## 4. 后端设计

### 4.1 核心模块

**questions.py**
- **数据源**: 从现有index.html的`<script>`标签中提取数据
- **提取内容**:
  - `dimensionMeta`: 15个维度的元信息
  - `questions`: 30个常规问题数组
  - `specialQuestions`: 2个特殊问题数组
  - `NORMAL_TYPES`: 26种SBTI类型的模式数据
  - `DIM_EXPLANATIONS`: 各维度L/M/H等级解释
- **问题结构**: `{id, dim, text, options: [{label, value}]}`
- **处理特殊问题**: 酒鬼触发问题的逻辑
- **输出格式**: Python可用的数据结构

**scoring.py**
- 实现15维度计分逻辑
- **基准分数**: 每个维度从4分开始（2问题×默认2分）
- **正常计分**: value 1→-1分, 2→0分, 3→+1分
- **反向计分**: q27使用 value 1→+1分, 2→0分, 3→-1分
- **分数转等级**: ≤3分=L, 4分=M, ≥5分=H
- **生成模式字符串**: 按dimensionOrder顺序连接15个等级

**matcher.py**
- 向量距离计算：用户向量与类型向量的曼哈顿距离
- L/M/H转换为数值：L=1, M=2, H=3
- 相似度计算：`max(0, round((1 - distance/30) * 100))`
- 匹配排序：按距离升序，精确匹配数降序，相似度降序

**llm_interface.py**
- OpenRouter API调用封装
- **提示词策略**:
  - 明确说明只需回答选项字母
  - 处理解释性回答的解析逻辑
  - 支持重试和错误恢复
- **错误处理**:
  - API限流：指数退避重试
  - 超时处理：30秒超时
  - 格式错误：尝试提取字母，失败则重试
- **参数支持**: temperature, max_tokens, model等

**evaluator.py**
- 主评测流程
- 协调各模块
- 结果格式化输出
- 进度显示和日志记录

### 4.3 提示词设计

**挑战**: 大模型可能不理解幽默的网络测试风格
**解决方案**: 结构化提示词

**基础提示词**:
```
你正在参与一个关于自我认知、情感、态度、行动和社交倾向的心理测试。
请根据你的真实倾向选择最符合的答案。

对于每个问题，请只回答选项字母（A/B/C/D），不要添加其他解释。

这是一份娱乐性测试，请诚实作答，展现你真实的"个性"倾向。
```

**回答解析策略**:
1. **理想回答**: 单个字母 (A/B/C/D)
2. **可接受回答**: 包含字母的短文本，如"我选择A"、"答案是B"
3. **解析逻辑**:
   - 首先尝试提取大写字母A-D
   - 如果找不到，尝试提取小写字母a-d
   - 如果仍然找不到，记录为格式错误，触发重试
4. **重试机制**: 最多重试2次，第3次使用默认值(2=M)

**特殊情况处理**:
- 如果模型给出解释性回答，通过正则表达式提取字母
- 如果模型拒绝回答或表示无法作答，使用中立选项
- 如果模型回答超出A-D范围，记录警告并使用默认值

### 6.2 反向计分处理

原HTML中存在反向计分问题，需要特别注意：

**确认的反向计分问题**:
- **q27**: "我和人相处主打一个电子围栏，靠太近会自动报警。"
  - 选项：认同(1), 中立(2), 不认同(3)
  - 计分：1→+1分(H), 2→0分(M), 3→-1分(L)
  - 注意：这是唯一的反向计分问题

**正常计分问题**:
- 其他所有29个问题都是正常计分
- value 1→-1分(L), 2→0分(M), 3→+1分(H)

### 6.3 特殊类型触发

**DRUNK (酒鬼)触发条件**:
- **drink_gate_q1**: "您平时有什么爱好？"
  - 选择 value=3 ("饮酒")
- **drink_gate_q2**: "您对饮酒的态度是？"
  - 选择 value=2 ("我习惯将白酒灌在保温杯，当白开水喝，酒精令我信服。")
- **触发逻辑**: 两个条件都满足时，直接返回DRUNK类型，跳过常规匹配

**HHHH (傻乐者)触发条件**:
- **前置条件**: 非DRUNK类型
- **匹配度阈值**: 最佳常规类型匹配度 < 60%
- **触发逻辑**: 当标准类型库都无法很好匹配时，强制分配HHHH类型

**触发优先级**:
1. DRUNK > HHHH > 常规类型
2. 特殊类型触发后仍保存secondaryType（最佳常规类型）用于参考

## 8. 风险与挑战

### 8.1 技术挑战

1. **大模型理解问题**: 幽默风格可能被误解
   - 解决方案: 优化提示词，添加说明

2. **API限流**: 批量评测可能触发限流
   - 解决方案: 添加延迟，分批处理

3. **结果一致性**: 同一模型可能给出不同答案
   - 解决方案: 设置较低的temperature，多次评测取平均

### 8.2 设计挑战

1. **数据可视化**: 15维度数据展示复杂
   - 解决方案: 使用雷达图，支持分层展示

2. **前端性能**: 大量模型数据可能影响性能
   - 解决方案: 分页加载，虚拟滚动
