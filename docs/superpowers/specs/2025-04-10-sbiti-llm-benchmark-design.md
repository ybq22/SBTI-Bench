# SBTI大模型评测系统设计文档

**日期**: 2025-04-10
**项目**: SBTI-Bench
**目标**: 将真人SBTI测试系统改造为大模型SBTI评测基准

## 1. 项目概述

### 1.1 背景与目标

现有SBTI测试系统面向人类用户，提供26种人格类型评估。本项目将其改造为大模型评测基准，通过标准化测试评估不同AI模型的"人格"特征。

### 1.2 核心需求

- **评测系统**: Python脚本通过OpenRouter API评测单个大模型
- **批量处理**: Bash脚本支持多模型批量评测
- **数据存储**: JSON格式存储评测结果
- **前端展示**: 改造现有HTML，展示模型对比和排行榜
- **API统一**: 所有模型调用通过OpenRouter

## 2. SBTI规则分析

### 2.1 与MBTI的核心差异

- **15个维度** (vs MBTI的4个维度)
- **26种人格类型** (vs MBTI的16种)
- **三级评分系统**: L/M/H (低/中/高)

### 2.2 维度体系

**自我模型 (Self Model)**
- S1: 自尊自信
- S2: 自我清晰度
- S3: 核心价值

**情感模型 (Emotion Model)**
- E1: 依恋安全感
- E2: 情感投入度
- E3: 边界与依赖

**态度模型 (Attitude Model)**
- A1: 世界观倾向
- A2: 规则与灵活度
- A3: 人生意义感

**行动驱力模型 (Action Model)**
- Ac1: 动机导向
- Ac2: 决策风格
- Ac3: 执行模式

**社交模型 (Social Model)**
- So1: 社交主动性
- So2: 人际边界感
- So3: 表达与真实度

### 2.3 评测机制

- **30个常规问题** + 2个特殊问题（酒鬼触发）
- 每个问题1-3分，部分问题反向计分
- 维度分数转换为L/M/H等级
- 通过向量距离算法匹配最相似的SBTI类型

### 2.4 26种SBTI类型

包括但不限于：
- CTRL (拿捏者), BOSS (领导者), SHIT (愤世者)
- DRUNK (酒鬼), HHHH (傻乐者) - 特殊类型
- 以及其他21种类型，每种都有独特的15维度模式

## 3. 系统架构

### 3.1 整体结构

```
SBTI-Bench/
├── backend/                 # Python后端
│   ├── questions.py        # 问题数据提取
│   ├── scoring.py          # 计分逻辑实现
│   ├── matcher.py          # 匹配算法
│   ├── llm_interface.py    # OpenRouter API接口
│   ├── evaluator.py        # 主评测脚本
│   └── requirements.txt    # Python依赖
├── batch_scripts/          # 批量评测脚本
│   └── batch_eval.sh       # Bash批量脚本
├── frontend/               # 前端展示
│   ├── index.html         # 展示页面（改造现有）
│   ├── css/               # 样式文件
│   └── js/                # JavaScript逻辑
├── data/                  # 评测数据
│   └── results.json       # 评测结果
├── image/                 # 现有图片资源
├── .env                   # 环境配置
└── README.md              # 项目说明
```

### 3.2 数据流

```
用户输入模型列表
    ↓
bash批量脚本
    ↓
Python评测脚本 (逐个模型)
    ↓
OpenRouter API调用
    ↓
模型回答SBTI问题
    ↓
计算15维度得分
    ↓
匹配SBTI类型
    ↓
保存JSON结果
    ↓
前端展示页面读取JSON
    ↓
生成排行榜和对比图表
```

## 4. 后端设计

### 4.1 核心模块

**questions.py**
- 从原HTML提取30个问题
- 问题结构: `{id, dim, text, options: [{label, value}]}`
- 处理特殊问题（酒鬼触发）

**scoring.py**
- 实现15维度计分逻辑
- 处理反向计分问题（如q27）
- 分数转L/M/H等级转换
- 生成模式字符串

**matcher.py**
- 向量距离计算
- SBTI类型匹配算法
- 相似度计算

**llm_interface.py**
- OpenRouter API调用封装
- 提示词设计
- 错误处理和重试
- 支持自定义参数（temperature, max_tokens等）

**evaluator.py**
- 主评测流程
- 协调各模块
- 结果格式化输出

### 4.3 提示词设计

**挑战**: 大模型可能不理解幽默的网络测试风格
**解决方案**: 结构化提示词

```
你正在参与一个关于自我认知、情感、态度、行动和社交倾向的心理测试。
请根据你的真实倾向选择最符合的答案。

对于每个问题，请只回答选项字母（A/B/C/D），不要添加其他解释。

这是一份娱乐性测试，请诚实作答，展现你真实的"个性"倾向。
```

## 6. 技术实现细节

### 6.1 OpenRouter集成

**配置文件 (.env)**
```env
OPENROUTER_API_KEY=your_key_here
DEFAULT_MODEL=openai/gpt-4
DEFAULT_TEMPERATURE=0.7
DEFAULT_MAX_TOKENS=1000
```

**API调用特点**:
- 统一通过OpenRouter
- 支持多种模型（GPT-4, Claude, Llama等）
- 错误重试机制
- 限流处理

### 6.2 反向计分处理

原HTML中存在反向计分问题（如q27），需要特别注意：
- 正常问题: value 1→L, 2→M, 3→H
- 反向问题: value 1→H, 2→M, 3→L

### 6.3 特殊类型触发

**DRUNK (酒鬼)触发条件**:
- q1选择"饮酒"
- q2选择"酒精令我信服"

**HHHH (傻乐者)触发条件**:
- 标准类型匹配度<60%

## 8. 风险与挑战

### 8.1 技术挑战

1. **大模型理解问题**: 幽默风格可能被误解
   - 解决方案: 优化提示词，添加说明

2. **API限流**: 批量评测可能触发限流
   - 解决方案: 添加延迟，分批处理

3. **结果一致性**: 同一模型可能给出不同答案
   - 解决方案: 设置较低的temperature，多次评测取平均

### 8.2 设计挑战

1. **数据可视化**: 15维度数据展示复杂
   - 解决方案: 使用雷达图，支持分层展示

2. **前端性能**: 大量模型数据可能影响性能
   - 解决方案: 分页加载，虚拟滚动

## 9. 成功标准

1. **功能完整**: 能够评测多个模型并生成结果
2. **数据准确**: 计分逻辑与原系统完全一致
3. **展示清晰**: 前端页面直观展示对比结果
4. **易于使用**: 提供清晰的使用文档
5. **可扩展**: 易于添加新模型和新功能
